Authful

Authful was written as an internal app for MongoHQ to be used with our web application. It supports multiple accounts and could be made into a public 2FA service, although there is no frontend on it at this point. We use multiple accounts to allow for staging and testing setups that are seperate.

We are opensourcing this effort so that the community can have a 2FA solution that doesn't involve being locked in to vendor phone apps. We are accepting pull requests that add/imrove a UI for viewing and managing accounts and users as well as any bug fixes.

About

Authful is a 2fa engine. It behaves in the truest sense of SOA, and should be used as an API behind the scenes of your application. The purposes of Authful are:

  • Manage users for 2fa including:
    • enrolling / unenrolling users with either SMS or QR codes
    • validating user tokens
    • reseting tokens
    • managing recovery codes
  • Generate QR codes for (RFC 6238 - TOTP)[https://tools.ietf.org/html/rfc6238]

Requirements & Starting

  • Ruby 2.0.0-px
  • MongoDB - be sure your config/mongoid.yml file is configured correctly
  • Twilio Account with the following environmental variables set:
    • TWILIO_SID
    • TWILIO_TOKEN
    • TWILIO_FROM

Gem installation is a simple gem install bundler && bundle install

After installing the gems, you can run the application locally by running rackup -p 4000.

Testing the platform

Create a dummy account From bash, run:

rake db:seed

You will see output with tokens to use for auth and sample curl commands. Note: The tokens in the snippet below will not work on your dev setup. You will need to get them from your console output.

Account Token: 5i4ximidh3d7q3o2
   User Token: nsaxk2
  Current OTP: 734637

** Viewing QR Code **
http://authful.dev/qr/nsaxk2.png

** Enrolling a user **
curl -X POST http://authful.dev/api/users -H "Api-Token: 5i4ximidh3d7q3o2" -d email=<user-email> -d phone=<phone-number>

** Validating Code **
curl -X GET http://authful.dev/api/users/nsaxk2/validate -H "Api-Token: 5i4ximidh3d7q3o2" -d token=734637

** Unenrolling a user **
curl -X DELETE http://authful.dev/api/users/nsaxk2 -H "Api-Token: 5i4ximidh3d7q3o2"

Enrolling a user via app

curl -X POST http://authful.dev/api/users -H "Api-Token: <account-token>" -d email=<user-email>

Enrolling a user via sms

curl -X POST http://authful.dev/api/users -H "Api-Token: <account-token>" -d email=<user-email> -d phone=<phone-number>

View a user

curl -X GET http://authful.dev/api/users/<user-token> -H "Api-Token: <account-token>"

Requesting a code via SMS

curl -X GET http://authful.dev/api/users/<user-token>/send_sms -H "Api-Token: <account-token>"

Validating Code

curl -X GET http://authful.dev/api/users/<user-token>/validate -H "Api-Token: <account-token>" -d token=<token generated or recieved via sms>

Register a fallback SMS

curl -X PATCH http://authful.dev/api/users/<user-token>/fallback -H "Api-Token: <account-token>" -d phone=<phone number>

Send code to a fallback SMS

curl -X GET http://authful.dev/api/users/<user-token>/fallback -H "Api-Token: <account-token>"

Generate recovery codes

curl -X POST http://authful.dev/api/users/<user-token>/recovery_codes -H "Api-Token: <account-token>"

View remaining recovery codes

curl -X GET http://authful.dev/api/users/<user-token>/recovery_codes -H "Api-Token: <account-token>"

Validate recovery code

curl -X GET http://authful.dev/api/users/<user-token>/recovery_codes/<recovery-code> -H "Api-Token: <account-token>"

Unenrolling a user

curl -X DELETE http://authful.dev/api/users/<user-token> -H "Api-Token: <account-token>"
